## Active project: `stock_trade_demo`
This is a script-first A-share quantitative stock-selection and backtesting project with:
- a reusable strategy framework under `stock_trade_demo/strategies/`
- a shared backtest engine in `stock_trade_demo/backtest.py`
- Chan-theory factor generation in `stock_trade_demo/chan_factors.py`
- a Flask web app in `stock_trade_demo/web_app.py`
- an ECharts frontend in `stock_trade_demo/web/templates/index.html`
- several CLI and research scripts for comparison / optimization

## Main entrypoints
- Web app: `stock_trade_demo/web_app.py`
  - Run with: `python3 web_app.py`
  - Expected URL: `http://localhost:8080`
- Single-strategy CLI: `stock_trade_demo/choose_stock.py`
- Multi-strategy compare CLI: `stock_trade_demo/compare_strategies.py`
- Data conversion helper: `stock_trade_demo/convert_data.py`

## Core architecture
- `stock_trade_demo/strategies/base.py`
  - Shared strategy pipeline:
    - `prepare_data -> compute_factors -> apply_filters -> rank_stocks -> run`
  - Also holds strategy metadata and selection-reason helpers.
- `stock_trade_demo/strategies/original.py`
  - Original small-cap strategy.
- `stock_trade_demo/strategies/chan_enhanced.py`
  - Market-cap-rank-dominant strategy with normalized Chan tilt.
- `stock_trade_demo/strategies/chan_only.py`
  - Weighted rank blend of Chan rank and size rank.
- `stock_trade_demo/strategies/method_a.py`
  - Daily Chan pipeline aggregate with normalized tilt on size rank.
- `stock_trade_demo/strategies/quality_value.py`
  - Z-score-normalized linear multi-factor strategy.
- `stock_trade_demo/backtest.py`
  - Data loading, regime tagging, selection, take-profit logic, capital path, evaluation.
- `stock_trade_demo/web_app.py`
  - Preloads data and cached strategy results, exposes APIs, filters by date range.
- `stock_trade_demo/web/templates/index.html`
  - Strategy selector, factor panel, overview/train-test charts, holdings modal.

## Data and cache expectations
- Main dataset: `stock_trade_demo/stock_data.csv`
- Optional performance format: parquet generated by `convert_data.py`
- Project-local cache: `stock_trade_demo/.cache/`
- Benchmark cache: `stock_trade_demo/.cache/csi1000_monthly.csv`
- Method A may reference cached factor files under `.cache/chan_factors_v2/`

## Current frontend/backend behavior worth preserving
- The factor panel is driven by strategy metadata from `BaseStrategy.get_factor_metadata()`.
- Holdings explanations are driven by strategy-level `build_selection_reason(...)` methods.
- `selection_factor_breakdown` is the structured field used to explain why a stock was selected.
- Custom date ranges reset capital to 100k and compute a standalone period curve.
- The app also exposes benchmark comparison and holdings detail for the selected period.
