4. **当前本地缓存重叠日期上的价格比值全为 1.0**：
   - `nasdaq_daily.csv` vs `timing_etf/nasdaq_etf_daily.csv`
   - `sp500_daily.csv` vs `timing_etf/sp500_etf_daily.csv`

- **当前结论不是“美股数据有 fake”，而是“fake 暂未发现，但生产口径还没审干净”。**

**核心评分公式 + 约束**（在 `scripts/walk_forward_train.py` 顶部）：
```python
TRAINING_CUTOFF      = 2025-11-30
SCORE_WEIGHTS        = {recent_6m: 0.4, recent_1y: 0.3, full_pre_cutoff: 0.3}
MAX_DD_THRESHOLD     = 0.20    # 任一窗口 |maxDD|>20% ⇒ score=-inf
FULL_NAV_FLOOR_RATIO = 1.00    # candidate.full_pre_cutoff.final_nav 必须 >= 1.00*default
```

**Fallback-to-default 安全网**：grid 内没有任何 candidate 满足 floor ⇒ 写入 `tuned_params = DEFAULT_TUNED[sid]`、`fallback_to_default = True`。Web 端读取后等价于"不调参"，行为与 default 完全一致。

## 6. 重要"踩坑提示"（前几次踩过 / 容易踩）

1. **改 `web/templates/*.html` 后必须重启 Flask**，Jinja 不会 auto-reload（debug=False）。重启流程：
   ```
   PID=$(lsof -nP -iTCP:8080 -sTCP:LISTEN -t)
   kill $PID; sleep 3
   cd stock_trade_demo
   nohup /Users/fatcat/opt/anaconda3/bin/python web_app.py > /tmp/web_app_phase2.log 2>&1 &
   ```
   截图前 curl `http://localhost:8080/api/info` 等返回 200 再 Selenium。

2. **改 best_profile JSON 后必须 bump `_CACHE_VERSION`**（`web_app.py`，当前 = 12）+ 重启 web。否则 cache 不更新。

3. **walk-forward 跑 nasdaq/sp500 时会反复打印 akshare 失败 + Sina HTTP 456**。这是 normal——akshare/Sina 限流，会 fallback 到 `stock_trade_demo/.cache/timing_etf/*.csv`。不要"修"这个 warning。

4. **`DEFAULT_TUNED`（`scripts/walk_forward_train.py:72`）必须与 `timing/strategies.py` 各策略类 `__init__` 默认值严格一致**。若改了策略类默认值，必须同步改 `DEFAULT_TUNED`，否则 floor reference 错位会让 fallback 逻辑失效。

5. **`comparison` 脚本** `/tmp/quant_verify/compare_defaults_vs_bestprofile.py` 默认会同时跑 full panel（含 holdout）。要分清"训练区退步" vs "holdout 段 OOS 差异"，必须看拆分后的 `pre_cutoff` / `holdout` / `full` 三行。

6. **CLAUDE.md Rule 13 是高压线**：不允许把 holdout 任何指标反过来做 fit / floor / score weight。如果用户问"holdout 也能不能更好"，正解只有"推迟 cutoff"或"等更多数据"，不是改 score。

7. **`.claude/` 与 `stock_trade_demo/.claude/` 不要 commit**。`settings.local.json` 是本地 Claude 权限授予记录，包含路径偏好等本地信息。

8. **commit 不要 -A**。新增的 `data/` 目录（如有）和 `output/` 目录如非用户明确要保留，都不应进 git。
