## 四、2026-05-27 cache-miss 事故 retrospective

**事故根因文档**：`docs/cache_miss_regression_2026-05-27.md`

### 影响
- `/timing`（csi1000 / chinext / star50）+ `/us_timing`（macro_v32 / sp500）
  默认页一打开全部卡片显示 "—"，回测 API 全 400 `cache_miss`。
- 月度选股、`/live`、数据更新接口、所有离线产物未受影响。

### 根因（双层）
1. **直接**：前端 `collectParamsForStrategy` 把全部 slider 默认值显式回放进 URL；
   后端 `_TIMING_CACHE_DEFAULTS` 用字面 `!=` 比对，任何形态差异（浮点、bool 字符串）
   都会让 cache hit 判断失败 → load-only 改造禁用了现算兜底 → 直接 HTTP 400。
2. **深层**：Pillar 1 Step 6 验收只覆盖"显式无参 cache hit + 显式带参 cache miss"
   两条路径，没考虑"默认页加载也会带参（前端忠实回放 default）"这条用户高频路径。
   保护 6 的 TimingParams 只接管类型转换，没接管"该不该走 cache"，命中判断仍散落在
   两个 blueprint 里。
